# Alpha Qualification Guide

Use this guide when screening a Temporal TypeScript repo for the Fabrik design-partner alpha.

## Goal

Decide whether a repo is:

- `qualified`
- `qualified_with_caveats`
- `blocked`

The source of truth is the migration output generated by:

```bash
cargo run -p fabrik-cli -- migrate temporal <repo> --output-dir <dir>
```

## Required Artifacts

Review these files first:

- `migration-report.json`
- `migration-report.md`
- `workspace/alpha-qualification.json`
- `workspace/support-matrix.json`
- `workspace/trust-summary.json`

## How to Read the Verdict

- `qualified`: the repo fits the current alpha subset and passed the available migration gates
- `qualified_with_caveats`: the repo fits the subset, but replay or deployment evidence is still missing
- `blocked`: the repo is outside the supported subset or failed a trust/operational gate

## Blocker Categories

- `unsupported_api`
  Use when the repo depends on unsupported Temporal APIs, blocked payload conversion behavior, or compiler-subset gaps.
- `unsupported_packaging_bootstrap`
  Use when the worker bootstrap shape or generated packaging path is not compatible with the current migration pipeline.
- `unsupported_visibility_search_usage`
  Use when the repo depends on memo or search-attribute behavior that Fabrik does not yet support in alpha.
- `replay_trust_blocker`
  Use when replay or rollout validation fails against Fabrik-side histories.
- `operational_blocker`
  Use when deployment, poller registration, or other operator-facing prerequisites fail.

## Qualification Flow

1. Run the analyzer and packaging flow.
2. Read the alpha qualification verdict.
3. Check whether the blocker categories are empty.
4. If the repo is `qualified_with_caveats`, decide whether the remaining caveats are acceptable for a design-partner engagement.
5. If the repo is a primary alpha candidate, rerun with deployment and replay validation enabled.

## Primary vs Shadow Repo

- Primary alpha repo:
  Must package, deploy, and run one representative workflow path on Fabrik.
- Shadow qualification repo:
  Must analyze cleanly and produce a trustworthy blocker/remediation report, even if it is not deployed.
